## Architecture

- Theme: Vessel's actual code lives in this repo at root (`layout/`, `sections/`, `blocks/`, `snippets/`, `templates/`, `config/`, `locales/`, `assets/`) — imported from a theme export (.zip) the user downloaded from Admin → Online Store → Themes → Vessel → Edit code → Download, after both the Theme Access token and Shopify's "Connect from GitHub" flow failed to establish a live sync (see Known Constraints / Gotchas).
- This is a **one-time import, not a live sync**. Edits made here won't appear on the store until re-uploaded (see the open task in Work Packages WP2.1 for how). Conversely, any edits made in Shopify's theme editor after this export won't be reflected here until re-exported.
- Confirmed: Vessel's `product.json` template already includes `variant-picker`, `quantity`, and `buy-buttons` blocks in its default product section (`blocks/variant-picker.liquid` etc., referenced in `templates/product.json`'s `main` section, block `product-details`). This means the Roast Profile / Quantity selection UX needed for the subscription plan builder is handled natively — no custom Liquid was needed for that piece.
- `templates/index.json` (homepage) has been rebranded: hero copy, and the two "Featured collection" sections now point at real collection handles (`coffee-subscriptions`, `gifts`) instead of Vessel's demo "all" placeholder. Two empty "featured product" demo sections (no product assigned) were removed rather than left blank.
- Subscription logic (recurring billing, customer portal) lives in the subscription app (Skio, pending confirmation), integrated into the theme via app blocks/sections — not built custom in Liquid.
- Store data/catalog changes (products, collections, pricing, pages) go through the Shopify MCP connector, independent of theme choice.
- Navigation menus (main + footer) still need to be pointed at the real collections/pages — blocked on the MCP connector being reachable (it disconnects/reconnects intermittently this session).

## Conventions

Vessel's own conventions apply: Online Store 2.0 JSON templates, a `blocks/` directory (theme blocks, newer Shopify feature beyond Dawn's model) alongside `sections/` and `snippets/`. Match its existing block/section patterns for any further customization rather than introducing Dawn-style patterns.

## Environment & Setup

- Shopify store connected via Shopify MCP connector: **Brown Bag Coffee Club** (`brown-bag-coffee-club.myshopify.com`), Basic plan, USD, US/EDT
- MCP connector gives Claude scoped store access (products, collections, orders, customers, inventory, analytics, discounts, pages, and general Admin GraphQL) — no raw admin credentials exchanged. It has disconnected/reconnected several times this session; treat gaps as transient, not a sign the integration was removed.
- Theme Access app: attempted twice (two separately generated tokens), both rejected by Shopify with 401 on every endpoint tested (including the themes endpoint itself), against both the correct store domain and the token's originating store. Root cause undiagnosed — likely something about how the app installed. Not pursued further once the manual export path worked; worth revisiting if live theme sync becomes worthwhile.
- Shopify CLI is installed globally in this environment (`npm install -g @shopify/cli`) for when a working auth method (Theme Access token, or `shopify auth login` interactively) becomes available.

## Known Constraints / Gotchas

- Claude Code can write/customize theme code but cannot perform Shopify admin actions directly (store creation, app installation, payment/tax setup) — those require the user to act in the Shopify dashboard, or to grant API/CLI access.
- Reference site's standard (non-promotional) pricing and full FAQ content are JS-rendered and weren't retrievable via WebFetch's HTML summarizer — worked around by pulling the raw catalog from Shopify's public `/products.json` endpoint instead (see Product Catalog below), which returns real prices directly.
- Atlas's live store contains many near-duplicate products (ad-campaign-specific variants, e.g. tagged `facebook-2023`, plus a straight "10% off" line) with different prices for what's otherwise the same product. The pricing table below uses the variant that appeared identically across multiple non-discounted listings, treated as their standard baseline — not a guarantee it's their exact current sticker price.
- Atlas does not sell specific named single-origin coffees as separate SKUs — the origin rotates monthly and is communicated via the postcard/tasting notes insert, not as a selectable product option. The purchasable variant axis is **Roast Profile** (Smooth & Balanced / Bright & Expressive), not origin. Our v1 catalog mirrors this structure; origin copy is placeholder/rotating content, not a product field.
